Introduction

Welcome to JSRogue! This is a roguelike game where your objective is to descend to the bottom of the dungeon and steal a chest of treasure. Many enemies will try to cut your quest short, but fortunately the dungeon is scattered with items you can use to help you on your way.

The unique aspect of JSRogue is that you don't control your character by pressing keys or clicking the mouse - you write javascript code which then controls your character! Hopefully the game serves as a fun way for you to learn some javascript and fundamental programming concepts. Before playing the game, you should read the rest of this guide so you can start writing some code.

Game Basics

Playing the game involves writing code in the Text-Area of the code section under the 'Code' tab. Then go the the 'Game' tab and press a button. If you wrote correct code for making your character perform some action it should be performed. To help you start writing code I've provided some pre-made code snippets in the section underneath the text area in the 'Code' tab. To use them simply click on the part of the text area where you want the pre-written code placed, then press the 'paste me' button underneath the code snippet you want to use a that point. You should also mouse over the code snippets to read a detailed description of what they do. Note that your code is saved automatically, though your game progress is not: the former is much more imortant than the latter!

You can look at different parts of the current level while playing by clicking and dragging the play area left or right. Also, there is a secondary coding area underneath the play-area - you can code there if you don't want to click away from the play-area, but the main code-tab has the pre-written code snippets.

Coding Basics

JSRogue is turn based, and each time you perform an action by pressing any key on the game page after writing correct code, all the lines of code you've entered into the text area will be executed, starting from the top and ending at the bottom.

But how do you write this code? As a simple example, consider the following code-snippet which maker your character move one step north: 

this.step('N')

What exactly is going on here? First of all, we begin by referring to the player character by using the keyword 'this'. Now, the player character is a JavaScript object, and has both a selection of attributes we can access, examine and modify, and methods we can use to perform some action. You can think of attributes as properties of an object, such as its height or weight, and methods as actions associated with the object. The way we access an object's properties and methods is by referring to the object (in this case using the 'this' keyword), then placing a full-stop after this reference (so we now have 'this.') and then referring to the property or the method we want to access after the full stop (which gets us 'this.step'). At this stage we now have a reference to the 'step' method of the player character object, but that isn't sufficient to actually step somewhere. We don't want to simply refer to this method - we want to use it! To do so, we must place a pair of brackets after our reference to the method, which gets us 'this.step()'. With some methods this is all we need to do; however, we cannot simply step - we must step in a direction. This method, like many others, requires us to supply a parameter between the braces, in this case a direction. The method is such that it recognises 'N' as north, so we place it between the braces, remembering to keep the speech marks and case, finally getting us 'this.step('N')', the code we started with. This pattern is central to the code you will write to control your character.

Return values

Consider the following code snippet: 

var strikeCheck = this.watchForStrike()

By now it should be clear what is going on to the right of the '=' sign in the code above: we are executing the 'watchForStrike' method of 'this'. It might also be apparent what 'watchForStrike' does: it checks if the player character is currently facing an incoming strike. But what's happening to the left of the '=' sign? What we're doing is assigning the return value of the execution of the watchForStrike method to the variable 'strikeCheck'. If the player is about to be hit, strikeCheck will hold the value true, otherwise it will hold the value false. We can then use this variable in the rest of our code. Note: the name of the vaiable is arbritrary, and you can use whater you find convenient.

'If' Statements

One such use of a return value stored in a variable is to decide what portion of the rest of our code should be executed. In Javascript, just as in many other programming langauges such as C, we can do this by means of an 'If' statement. This is illustrated in the following example: 

var strikeCheck = this.watchForStrike()
if (strikeCheck)
{

}
As before we store the return value of the call of watchForStrike in strikeCheck. But this time we actually use the variable for something - to check if we should execute some other instructions between the curly braces. We do this by placing the variable to be evaluated as true or false in the brackets after the 'if'. No intstructions are present in this example between the curly brackets, but if we, for instance, placed 'this.step('N')' between the braces, this line would be executed if the value of strikeCheck was true: that is, if the player was about to be struck. Multiple lines can be placed between the braces, and any code below the braces is executed after that inside the braces is. Otherwise, if strikeCheck is false, the code between the braces is skipped and the code after the braces, if present, is executed.

Arrays

Some method calls return a special typr of object: arrays. Any array is a container object which keeps other objects in a certain order. Consider the following example: 

var enemies = this.lookForEnemies();
if(enemies.length>0)
{
var firstEnemy = enemies[0]
}

In this first line we call the method lookForEnemies, which returns an array containing references to all the enemies, which we store in the variable 'enemies'. All arrays have a length attribute, so we can check if we've seen an enemy by checking if the length of the array stored in 'enemies' is greater than 0, which is what we do between the brackets after the 'if'. If that is true, then, as pointed out above, the code between the curly brackets will be executed. In that case we will likely want to do something to one of the enemies we seen, so we get a reference to the first enemy seen and store it in 'firstEnemy'. We do this by accessing the first object in enemies, which we do by placing square brackets after the 'enemies' variable, enclosing the position of its first element. Positions are counted from 0, so to get the first element we put 0 between the square brackets. Though it is not included in the example, at that point we could use the variable 'firstEnemy' to do something to the first enemy we seen. We could pass it as a parameter to a method which makes the player character step towards the enemy, or shoot an arrow at it, for instance.

The 'Directions' array

A premade array is provided for you to use in the form of the 'Directions' object, referred to as Game.directions. This is best used with repeatedely performing an action which takes a direction as input, but where you don't want to repeat the code with different direction parameters. The example in the next section illustrates this.

'For' Loops

We might want to check what spaces next to us are free, so that, e.g., we can move there (to dodge an enemy attack, for instance). We could do this by putting 'this.canMove('N')', 'this.canMove('S')' etc. on seperate lines. But this is very clumsy. It is better to use the following idiom: 

for (direction in Game.directions)
{
if (this.canMove(direction))
 {

 }
}
The purpose of the outer 'for' statement followed by brackets and outer curly braces is to execute the code enclosed by the outer curly brackets repeatedely with a different direction value assigned to the variable 'direction' from the Game.directions array each time. This is done as many times as there are directions in Game.directions. To illustrate how to use the 'direction' variable, I have placed some inner code which checks if we can movde in the currently selected direction from Game.directions. You could place some code in the inner curly brackets which would be executed if that direction was free to move in. After the code has been executed with every different direction value in the Game.directions array, the rest of the code below is executed.

Note: The use of the variable name 'direction' is arbritrary - we could just as well have written 'for (blah in Game.directions)' instead, and use 'blah' as a variable.

To see a for loop used to execute a line of code a defined amount of times, consider the following code: 
for (counter = 0; counter < 3; counter++)
{
this.drinkPotion()
}
Before the first semicolon in the brackets after the 'for' we define a variable 'counter' and assign it the value 0. Between the first and second semicolon we say that we should stop executing the code in the curly braces once 'counter' stops being less than 3, while the code between the last semicolon and brackets means that each time we execute the code between the curly braces, we should increment 'counter': add one to it. This has the effect that we perform the code between the curly brackets 3 times, so we drink 3 potions. The code is executed once with 'counter' having a value of 0, then again with a value of 1, then 2, then it stops when the value becomes 3. As an aside, we could have used 'counter' as a variable within the code in the curly braces, and this is sometimes very usefull, but not neccessary in this example

'While' Loops

'While' loops are similar to 'for' loops in that they can be used to execute a section of code repeatedely. Specifically, they can be used to execute a section of code until a pre-defined condition is met. Consider the following example: 

while (this.currHealth<500)
{
this.drinkPotion()
}
The code between the curly braces in this example will be executed until the statement 'this.currHealth<500' stops being true: that is, until the players current health exceeds 499.

Method List


Note: You can can only perform one of the following actions a turn: moving, physically attacking an enemy, raising your shield. Firing arrows, quaffing potions, looking for enemies and so on can be performed as many times as you wish.

this.exploreMap()
Function: This will move you towards the centre of each room in the map, one step each turn. After you've used this to go to the centre of each room, executing it will do nothing.

Parameters: None.

this.isLevelExplored()
Function: This returns true if you have used the exploreLevel method to explore every room in the level; false otherwise.

Parameters: None.

this.drinkPotion()
Function: This drinks a potion, healing 30 of your hit points.

Parameters: None.

this.shootArrowAt()
Function: This shoots an arrow at an enemy supplied as a parameter.

Parameters: The enemy to be shot.

this.descendSpottedStairs()
Function: This will move you towards a set of stairs you've seen, and descend them once you're standing on them.

Parameters: None.

this.watchForStrike()
Function: This returns true if a minotaur is about to strike you with his club; false otherwise.

Parameters: None.

this.raiseShield()
Function: This raises your shield to reduce the damage you take from incoming attacks for the next turn. If you have a mithril shield, you will also deflect arrows back at the enemy who shot them.

Parameters: None.

this.canMove()
Function: This checks if you can move in the direction passed as parameter next turn.

Parameters: A direction taken from the Game.directions object, or supplied directly as one of: 'N','NE','E','SE','S','SW','W','NW'. Remember to use the speech marks if supplying the direction directly.

this.step()
Function: This attempts to move one step in a direction supplied as a parameter.

Parameters: A direction taken from the Game.directions object, or supplied directly as one of: 'N','NE','E','SE','S','SW','W','NW'. Remember to use the speech marks if supplying the direction directly.

this.pickUp()
Function: This picks up any item lying on the ground. If the item is worse than one you already own, you will leave it.

Parameters: None.

this.lookForEnemies()
Function: This returns an array containing references to all enemies you've seen.

Parameters: You can supply an optional parameter consisting of the name of the type of enemy you want to look for. For instance, if this.lookForEnemies('Feral Dog') was executed, all enemies which were not Feral Dogs would be absent from the returned array. Remember to preserve the case and speech marks.

this.lookForItems()
Function: This returns an array containing references to all items you've seen.

Parameters: You can supply an optional parameter consisting of the name of the type of item you want to look for. For instance, if this.lookForEnemies('Potions') was executed, all items which were not Potions would be absent from the returned array. Remember to preserve the case and speech marks.

Miscellanea


'Target'

Signifies the direction in which an adjacent enemy is looking. If they are looking away, you can sneak past them.

'Player'

This is you!

'Stairs'

Descend these to progress to the next level. You can do so using the descendSpottedStairs method.

'Door'

Walk into these to open them.

Items


'Potions'

Pick these up to collect them. Drink them using the drinkPotion method if your health runs low. You can do this as many times a turn as you like, provided you have the potions to spare.

'Arrows'

Pick these up to collect them. You can shoot them at enemies as many times a turn as you like, provided you have the arrows to spare.

'Treasure'

Pick this up to win the game!

'Iron Sword'

Pick it up to improve your melee attack.

'Mithril Sword'

Pick it up to improve your melee attack even further!

'Copper Shield'

Pick it up to improve the effectiveness of your raised shield.

'Mithril Shield'

Pick it up to improve the effectiveness of your raised shield even further, and for your raised shield to deflect incoming arrows back at your enemies!

'Long Bow'

Pick it up to improve the effectiveness of your arrow damage.

'Mithril Bow'

Pick it up to improve the effectiveness of your arrow damage even further!

Enemies


'Feral Dog'

The weakest enemy in the game, but they come in packs.

'Orc'

A moderately strong melee fighter.

'Archer'

This enemy will shoot 10 arrows at you, one a turn, and then melee you.

'Orc Warrior'

This Orc fights with a sword and mithril shield at the same time, so has a high defense and can bounce arrows you shoot back at you.

'Minotaur'

This is the strongest enemy you'll face in the game. Fortunately, it has a weakness. Minotaurs need to lift their club to strike you. You can check for this in your code by calling watchForStrike; if you're about to be struck, you can then dodge by stepping somewhere else.

Tips

  • Build your code incrementally. That way, if you make a mistake and your code no-longer runs, you know the problem is with the last piece if code you wrote. Don't try to write all the behaviour for your character at once - you don't need to.
  • If you want to stop code from executing while you type, simply type a line of gibberish to induce an error in the code, and delete it when you wnt to execute the code again.
  • Remeber to use speech marks to encolse the name of an object or direction you want to use as a parameter, and to use the correct case. Variables, on the other hand, aren't enclosed in speechmarks.
  • Use double backspaces in your code - // - to preface comments. The game won't attempt to execute these, so you can use them to write handy notes. Or to temporarily negate lines of code you don't want to delete.
  • Play around with the code! The game is deliberately farely short, so you don't lose to much progress if you die because of this
  • Have Fun!
